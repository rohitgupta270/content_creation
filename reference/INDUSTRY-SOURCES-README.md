## Recommended Workflow

### Week 1: Setup (60 minutes)
1. Read `sources-quick-reference.md` (5 min)
2. Read `curated-industry-sources.md` (15 min)
3. Subscribe to top 5 per category using `sources-urls-for-subscriptions.txt` (20 min)
4. Set up Feedly or your preferred aggregator (15 min)
5. Create weekly review habit on calendar (5 min)

### Daily: Monitor (15-20 minutes)
1. Scan top 4-5 sources (The Batch, FinTech Inside, PhocusWire, + 1 PM source)
2. Look for strong signals
3. Note anything interesting for weekly review

### Weekly: Log & Brainstorm (45 minutes)
1. Deep read 1-2 sources per category
2. Log signals in `trend-signals-YYYY-MM-DD.md`
3. Identify patterns across sources
4. Ask Claude: "Here's a signal from [source]. What's the PM angle for a post?"

### Monthly: Review & Plan (1 hour)
1. Review all captured signals from past month
2. Identify top 3-5 content ideas
3. Ask Claude to develop these into posts
4. Assess which sources were most valuable

## Key Principles

1. **Consistency beats depth** — 15 min/day is better than 3 hours on Sunday
2. **Patterns matter more than signals** — When multiple sources discuss same topic, that's your signal
3. **Signals → Problems → Posts** — Every trend should ask "What PM problem does this solve?"
4. **Quality over quantity** — 20 high-signal sources beat 100 mediocre ones
5. **Build with sources** — Reference original research, link back, build credibility

## Integration with Content Creation

### The Pipeline

```
FIND (Industry Sources)
    ↓
CAPTURE (Trend Signals Template)
    ↓
ANALYZE (What's the PM angle?)
    ↓
IDEATE (Viral replication / Trend surfing / Pain point)
    ↓
CREATE (Post with source attribution)
    ↓
PUBLISH (Build authority through citations)
```

### Example

**Signal:** PhocusWire publishes on NDC adoption accelerating in Asia-Pacific airlines

**Capture:** Log in trend-signals-YYYY-MM-DD.md with category=Travel, why it matters, PM angle

**Analyze:** "This is about distribution control and customer data ownership — a discovery and prioritization problem"

**Ideate:** Ask Claude: "Turn NDC adoption into a viral replication post. The angle: distribution shifts are discovery problems"

**Create:** Draft post that uses viral replication format + PM perspective + reference to PhocusWire

**Publish:** Post mentions source, builds trust, drives backlinks
